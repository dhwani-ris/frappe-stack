# /frappe-stack:init

First command run on a new project. Idempotent — safe to re-run when a token expires (DeployControl tokens are 1-hour) or after rotating the Frappe API key.

Nothing gets installed on the Frappe site. The plugin uses Frappe's stock REST API with token authentication.

## Before you run this

Make sure the [session-start protocol](../skills/process/session-start/SKILL.md) is complete — most importantly, `Security_DRIS.md` must be attached to your AI session.

## What it does

1. **Verify the site is reachable.** `curl <site>/api/method/ping` returns `pong`.
2. **Confirm version.** Refuse if Frappe < v15.
3. **Prompt for Frappe API key + secret.** Generated by you on the Frappe Desk: User → API Access → Generate Keys. Stored in your OS keychain (macOS Keychain / Windows Credential Manager / `pass` / etc.). Only a *reference* to the keychain entry lands in the local config file.
4. **Prompt for DeployControl GitHub token.** A 1-hour, repo-scoped token issued via DeployControl (the platform.2010 GitHub Token Generator). See the [DeployControl runbook](../docs/operators/deploy-control-tokens.md) for how to generate one. Also stored in OS keychain.
5. **Configure local plugin state** at `.frappe-stack/config.json`:
   ```json
   {
     "sites": {
       "staging": {
         "url": "https://staging.example.com",
         "api_key_secret_ref": "<keychain entry id>",
         "is_production": false
       },
       "prod": {
         "url": "https://prod.example.com",
         "api_key_secret_ref": "<keychain entry id>",
         "is_production": true
       }
     },
     "config_repo": {
       "url": "https://github.com/dhwani-ris/<repo>",
       "local_path": "/path/to/local/checkout",
       "branch": "main",
       "github_token_ref": "<keychain entry id>",
       "github_token_source": "deploy-control",
       "github_token_expires_at": "2026-05-05T15:00:00Z"
     }
   }
   ```
6. **Smoke-test:** call `GET /api/resource/DocType?limit_page_length=1` on the site; expect `200 OK`. Logs the test to `.frappe-stack/audit.jsonl`.

## Arguments

- `<site-url>` — required. Your Frappe site URL, e.g., `https://staging.example.com`.
- `--prod` — flag. Marks this as the production site. Without it, defaults to `staging`.

## Refuses if

- The site is < Frappe v15 (the plugin uses v15+ REST behaviors).
- The provided user lacks `System Manager` (the API key inherits the user's roles; without `System Manager`, most resource creates fail).
- Adding a `--prod` site when staging isn't configured yet (force order: staging first, prod second).
- The DeployControl token has already expired. Generate a new one and re-run.
- The repo URL doesn't start with `https://github.com/dhwani-ris/` (DeployControl tokens are scoped to that org).

## Examples

```text
# First-time setup
/frappe-stack:init https://staging.example.com

# Add prod after staging is happy
/frappe-stack:init https://prod.example.com --prod

# Re-run after the DeployControl token expires (every 1 hour during long sessions)
/frappe-stack:init https://staging.example.com

# Re-run after rotating the Frappe API key
/frappe-stack:init https://staging.example.com
```
